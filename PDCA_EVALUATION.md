# アイデアPDCA評価: Context-Aware Workspace Automator (CAWA)

## 1. アイデア概要
ユーザーの現在の作業コンテキスト（開いているファイル、ブラウザのタブ、Slackの会話、カレンダーの予定）をリアルタイムで解析し、次に必要なアクションを「予測的」に準備・実行するデスクトップOS統合型AIエージェント。

- **コア体験**: 「あ、それやろうと思ってた」を先回りして実行されている感覚。
- **ターゲット**: 高度なコンテキストスイッチが頻発するナレッジワーカー、エンジニア、PM。

## 2. 評価スコア (0.0 - 10.0)

| 評価項目 | スコア | 理由 |
| :--- | :---: | :--- |
| **市場性 (Marketability)** | 8.5 | AIエージェントの飽和が進む中、「実行」ではなく「予測と準備」に特化したツールは空白地帯。 |
| **実現可能性 (Feasibility)** | 7.0 | OSレベルの監視とLLMの低遅延推論が必要。既存のOpenClawやMCPを活用すればMVPは可能。 |
| **差別化 (Differentiation)** | 9.0 | チャットUIではなく、OSのバックグラウンドで動作し、UIを適応させる（Adaptive UI）点が斬新。 |
| **収益性 (Profitability)** | 8.0 | 高度な効率化ツールとして、月額$20+の高単価サブスクリプションが期待できる。 |
| **ビジュアル性 (Visual Appeal)** | 8.5 | 「AI感のない」ガラスモフィズムやミニマルなインジケーターを採用することで、プロ品質を演出可能。 |
| **総合スコア** | **8.2** | **基準（7.5）クリア。プロトタイプ作成に移行。** |

## 3. PDCA評価の詳細
- **Plan**: MacOS/Windowsのメニューバーに常駐し、アクティブなウィンドウのメタデータを収集。
- **Do**: ユーザーがコードを書き始めたら、関連するドキュメントをサイドバーに「静かに」表示し、テストランナーを準備。
- **Check**: ユーザーがその準備されたリソースを利用したかをトラッキングし、予測精度を学習。
- **Act**: ユーザーが不要とした通知パターンを削減し、より「透明な」支援へ進化。
